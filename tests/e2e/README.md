# E2E Tests

Playwright E2E test suite for ExifCleaner. Tests run against the compiled Electron app.

## How to Run

```bash
# Full suite (compile first)
yarn compile && yarn test:e2e

# Single spec file
yarn compile && npx playwright test app-launch.spec.ts

# With visible browser (headed mode)
yarn compile && npx playwright test --headed

# View HTML report
npx playwright show-report
```

## How to Add a Test

1. Create `tests/e2e/your-feature.spec.ts`
2. Import helpers: `launchApp`, `closeApp` from `./helpers/app_launcher`
3. Use `createFixtureDir()` from `./helpers/fixture_copier` for any file operations
4. Use `waitForProcessing()` from `./helpers/wait_for_processing` after triggering file processing
5. Use `assertMetadataStripped()` from `./helpers/metadata_assertions` to verify file output
6. Follow AAA (Arrange-Act-Assert) pattern
7. Use ARIA role selectors: `window.getByRole()`, `window.getByText()`
8. Never pass original fixture paths to the app -- always copy to temp dir first

## Fixture Conventions

- Fixtures live in `tests/e2e/fixtures/`
- Generated by `generate_fixtures.ts` using bundled ExifTool
- NEVER pass original fixture paths to the app -- always copy to temp dir first
- To regenerate: `npx tsx tests/e2e/fixtures/generate_fixtures.ts`

## Helper Reference

| Helper | Purpose |
|--------|---------|
| `app_launcher.ts` | Launch/close Electron app for tests |
| `fixture_copier.ts` | Copy fixtures to temp dir, cleanup |
| `metadata_assertions.ts` | Read EXIF via ExifTool, assert metadata stripped |
| `wait_for_processing.ts` | Wait for file processing to complete in UI |

## Architecture

Each spec follows the same pattern:

- `beforeEach`: Launch app, set up console error monitoring
- `afterEach`: Check for unexpected console errors, close app
- Tests use `app.evaluate()` for main process operations (nativeTheme, BrowserWindow)
- Tests use `window.evaluate()` for renderer DOM operations
- File processing tests use IPC to send files (`file-open-add-files` channel)

The app is compiled once before the suite runs. Each test launches a fresh Electron instance for isolation.
